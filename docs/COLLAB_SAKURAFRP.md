# 樱花 FRP（SakuraFrp）协作穿透指南

用 **一条** 樱花隧道把本机 **3888** 暴露出去即可。网页和协作 WebSocket 共用同一地址（协作走 `/collab` 路径，由 Vite 转发到本机 1234）。

---

## 原理（为什么只要一条隧道）

| 组件 | 本机端口 | 同事访问 |
|------|---------|---------|
| 编辑器网页 | 3888 | `https://你的隧道地址/` |
| 协作 WebSocket | 1234（仅本机） | `wss://你的隧道地址/collab` |

同事**只打开你给的链接**，不用单独记协作端口。设置里填一次公网地址，软件会自动拼出 `…/collab`。

---

## 1. 准备

1. 注册 [SakuraFrp](https://natfrp.com/) 并完成实名
2. 安装 **启动器** 或 **frpc**
3. 本机可运行 `start-with-sakurafrp.bat`

---

## 2. 创建 **一条** 隧道

| 选项 | 值 |
|------|-----|
| 隧道类型 | TCP（或 HTTP + 自有域名） |
| 本地 IP | `127.0.0.1` |
| 本地端口 | **3888** |

可选：开启 **自动 HTTPS**（推荐，避免浏览器拦截 WebSocket）。

**不要**再单独映射 1234；1234 只在本机跑协作服务，由 3888 代理。

---

## 3. 启动

1. 双击 **`start-with-sakurafrp.bat`**（协作 1234 + 前端 3888）
2. 樱花启动器里 **只开这一条隧道**
3. 从日志复制公网地址，例如：`https://cn-xx.natfrp.cloud:51906`

frpc 示例（仅一个隧道 ID）：

```powershell
frpc_windows_amd64.exe -f <访问密钥>:<隧道ID>
```

---

## 4. 编辑器设置

1. **设置 → 多人协作 → 樱花 FRP**
2. 粘贴上面的 **公网访问地址**
3. **套用 Sakura FRP** → **保存协作设置**
4. 画布 → **开始协作** → **复制邀请链接**

自动写入示例：

- 邀请链接地址：`https://cn-xx.natfrp.cloud:51906`
- 协作 WebSocket：`wss://cn-xx.natfrp.cloud:51906/collab`

---

## 5. 自检

| 检查 | 方法 |
|------|------|
| 本机 | `http://127.0.0.1:3888` 能打开 |
| 穿透 | 樱花公网地址能打开同一页面 |
| 协作 | 开始协作后状态为「已就绪」或「N 人在线」 |

---

## 常见问题

**以前文档说两条隧道？**  
早期没做 `/collab` 代理，才要分别映射 3888 和 1234。现在已合并，**一条即可**。

**同事还要填 WebSocket 吗？**  
不用。他们打开邀请链接；只有你（主机）在设置里填公网地址。

**frpc 报无法连接本地服务**  
先运行 `start-with-sakurafrp.bat`，确认本机 3888 已监听，再开隧道。
